# FinAgent Pro 部署指南

## 系统要求

### 硬件要求
- **CPU**: 4核及以上
- **内存**: 8GB及以上（推荐16GB）
- **硬盘**: 20GB可用空间
- **网络**: 可访问国内网络（无需VPN）

### 软件要求
- **操作系统**: Windows 10/11, macOS, Linux
- **Python**: 3.10+
- **Node.js**: 18+
- **Git**: 最新版本

---

## 快速部署

### 1. 克隆项目

```bash
git clone <项目仓库地址>
cd finagent-pro
```

### 2. 后端部署

#### 2.1 创建虚拟环境

```bash
cd backend
python -m venv venv

# Windows
venv\Scripts\activate

# macOS/Linux
source venv/bin/activate
```

#### 2.2 安装依赖

```bash
pip install -r requirements.txt
```

#### 2.3 配置环境变量

创建 `.env` 文件：

```env
# 大模型API配置
DEEPSEEK_API_KEY=your_deepseek_api_key
ZHIPU_API_KEY=your_zhipu_api_key

# 服务配置
HOST=0.0.0.0
PORT=8000
DEBUG=false

# 缓存配置
CACHE_DIR=./cache
```

#### 2.4 启动服务

```bash
python main.py
```

服务将在 http://localhost:8000 启动

API文档地址: http://localhost:8000/docs

---

### 3. 前端部署

#### 3.1 安装依赖

```bash
cd frontend
npm install
```

#### 3.2 配置API地址

编辑 `src/App.tsx`，修改API地址：

```typescript
const API_BASE_URL = 'http://localhost:8000';
```

#### 3.3 启动开发服务器

```bash
npm start
```

前端将在 http://localhost:3000 启动

## 生产环境部署

### Docker部署（推荐）

#### 1. 构建镜像

```bash
# 后端镜像（从项目根目录构建）
docker build -t finagent-backend -f Dockerfile.backend .

# 前端镜像（从项目根目录构建）
docker build -t finagent-frontend -f frontend/Dockerfile .
```

#### 2. 使用Docker Compose

创建 `docker-compose.yml`:

```yaml
version: '3.8'

services:
  backend:
    build: ./backend
    ports:
      - "8000:8000"
    environment:
      - DEEPSEEK_API_KEY=${DEEPSEEK_API_KEY}
      - ZHIPU_API_KEY=${ZHIPU_API_KEY}
    volumes:
      - ./cache:/app/cache
      - ./chroma_db:/app/chroma_db
    restart: unless-stopped

  frontend:
    build: ./frontend
    ports:
      - "3000:80"
    depends_on:
      - backend
    restart: unless-stopped
```

启动服务：

```bash
docker-compose up -d
```

> **注意**: 首次部署需执行 `alembic upgrade head` 初始化数据库

### 云服务器部署

#### 阿里云/腾讯云部署

1. **购买云服务器**（推荐配置：4核8GB）

2. **安装Docker**

```bash
curl -fsSL https://get.docker.com | sh
sudo systemctl enable docker
sudo systemctl start docker
```

3. **部署应用**

```bash
# 克隆项目
git clone <项目仓库地址>
cd finagent-pro

# 配置环境变量
cp .env.example .env
# 编辑 .env 文件，填入API密钥

# 启动服务
docker-compose up -d
```

4. **配置Nginx反向代理**

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /api {
        proxy_pass http://localhost:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 配置说明

### 大模型API配置

#### DeepSeek V3

1. 访问 https://platform.deepseek.com
2. 注册账号并创建API Key
3. 在 `.env` 文件中配置 `DEEPSEEK_API_KEY`

#### 智谱GLM-4（备选）

1. 访问 https://open.bigmodel.cn
2. 注册账号并创建API Key
3. 在 `.env` 文件中配置 `ZHIPU_API_KEY`

### 数据源配置

系统使用AKShare获取港股数据，无需额外配置。

如需使用其他数据源，修改 `backend/services/market_data.py`。

### 知识库配置

系统启动时会自动初始化知识库。

如需添加自定义知识，编辑 `backend/knowledge/finance_kb.py` 中的 `_init_knowledge` 方法。

## 故障排查

### 常见问题

#### 1. 后端启动失败

**问题**: `ModuleNotFoundError: No module named 'xxx'`

**解决**: 
```bash
pip install -r requirements.txt
```

#### 2. 前端无法连接后端

**问题**: `Network Error` 或 CORS 错误

**解决**:
- 检查后端服务是否启动
- 检查 `App.tsx` 中的 API 地址配置
- 检查防火墙设置

#### 3. 股票数据获取失败

**问题**: `获取股票数据失败`

**解决**:
- 检查网络连接
- 确认AKShare是否安装正确
- 查看后端日志获取详细信息

#### 4. 大模型API调用失败

**问题**: `API调用失败` 或 `Rate Limit Exceeded`

**解决**:
- 检查API Key是否正确
- 确认账户余额充足
- 检查API调用频率限制

## 安全建议

1. **API密钥管理**
   - 不要将API密钥提交到Git仓库
   - 使用环境变量或密钥管理服务
   - 定期轮换API密钥

2. **访问控制**
   - 生产环境配置HTTPS
   - 使用防火墙限制访问IP
   - 配置Nginx反向代理

3. **数据安全**
   - 定期备份数据
   - 敏感数据加密存储
   - 限制数据库访问权限
